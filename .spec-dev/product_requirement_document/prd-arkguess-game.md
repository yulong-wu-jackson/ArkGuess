## 执行摘要

ArkGuess 是一款基于 Web 的「猜猜我是谁」数字棋盘游戏，支持明日方舟等主题的角色图片库。玩家可在可配置的网格上进行双人对战，通过标记角色来进行经典的猜人游戏。项目将通过 GitHub Actions 部署到 GitHub Pages。

---

## 问题陈述

### 问题描述
- 「猜猜我是谁」是一款经典的双人推理游戏，但目前缺乏便捷的数字化版本
- 玩家希望能使用喜爱的游戏角色（如明日方舟干员）来玩这款游戏
- 实体版游戏需要购买特定产品，无法自定义角色

### 影响
- 粉丝社区需要一个便捷、免费、可自定义的数字版本
- 可增强游戏社区互动，提供新的娱乐方式

### 现状
- 玩家只能使用官方实体版猜猜我是谁
- 或手动制作角色卡片，操作繁琐
- 缺乏支持自定义主题的数字化解决方案

## 目标与成功指标

### 主要目标
- [ ] **核心游戏功能**：实现完整的双人猜猜我是谁游戏流程
- [ ] **主题系统**：支持多主题图片库，首期支持明日方舟
- [ ] **易用性**：提供简洁直观的用户界面，支持中文
- [ ] **可部署**：通过 GitHub Actions 自动部署到 GitHub Pages

### 成功指标 (KPIs)
| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 页面加载时间 | < 3秒 | Lighthouse 性能测试 |
| 游戏初始化时间 | < 1秒 | 从选择完成到游戏开始 |
| 首次游戏启动时间 | < 30秒 | 从访问网站到进入游戏界面 |
| 标记操作响应时间 | < 100ms | 点击到视觉反馈 |
| 移动端兼容 | 100% 功能可用 | 手动测试主流设备 |

### 非目标
- **不包含**：在线多人联机（需服务器）
- **不包含**：用户账户系统
- **不包含**：自动AI对手
- **不包含**：跨设备同步（v1版本）

## 目标用户

### 主要用户画像
**游戏爱好者** — 明日方舟玩家及其他游戏社区成员

- **身份**：18-35岁游戏玩家，熟悉明日方舟等手游
- **需求**：使用喜爱的角色进行休闲娱乐
- **痛点**：缺乏便捷的数字化猜猜我是谁游戏
- **成功标准**：能快速开始游戏，界面简洁易用

### 次要用户
- **桌游爱好者**：希望数字化经典桌游
- **主题扩展者**：希望添加其他游戏/动漫主题

### 必须实现 (P0)

#### 主题选择
- [ ] **主题浏览** — 显示所有可用主题文件夹
  - 验收标准：读取 `/public/images/` 下的所有主题目录

- [ ] **网格大小选择** — 用户可选择 2x2 到 7x7 的网格
  - 验收标准：提供下拉菜单或滑块，限制范围 2-7

#### 随机模式
- [ ] **随机分配角色** — 从主题库随机选择角色填充网格
  - 验收标准：每个格子显示不同角色，不重复

- [ ] **角色展示** — 每格显示角色图片及名称
  - 验收标准：图片清晰可见，名称在图片下方

#### 自选模式
- [ ] **角色选择界面** — 展示主题内所有角色供多选
  - 验收标准：网格形式展示，点击可选中/取消

- [ ] **数量验证** — 验证选择数量匹配网格大小
  - 验收标准：选择数量不足时禁用开始按钮
  - 边界条件：若主题角色总数 < 网格大小²，限制可选网格上限并提示

#### 游戏机制
- [ ] **双方标记工具** — 提供红方(X)和蓝方(O)标记工具
  - 验收标准：清晰显示当前选中的工具

- [ ] **标记操作** — 点击角色进行标记/取消标记
  - 验收标准：首次点击添加标记，再次点击同类标记则取消

- [ ] **结束游戏** — 提供结束游戏按钮
  - 验收标准：点击后显示确认对话框，确认后返回主题选择页

### 应该实现 (P1)

- [ ] **响应式布局** — 适配桌面和移动设备
  - 验收标准：在 375px 至 1920px 宽度下正常显示

- [ ] **图片懒加载** — 优化大量图片的加载性能
  - 验收标准：仅加载视口内的图片

- [ ] **动画反馈** — 标记操作时的视觉反馈
  - 验收标准：标记添加/移除有平滑过渡动画

### 可以实现 (P2)

- [ ] **深色模式** — 支持明暗主题切换
  - 验收标准：自动跟随系统设置或手动切换

- [ ] **键盘快捷键** — 支持键盘操作
  - 验收标准：1/2 切换标记工具，Esc 结束游戏

## 用户体验

### 核心用户流程

#### 流程1：随机模式
1. 用户访问网站，进入主题选择页面
2. 选择主题（如：明日方舟）
3. 选择网格大小（如：4x4）
4. 选择"随机模式"
5. 系统随机分配 16 个角色到网格
6. 进入游戏界面，默认选中红方(X)工具
7. 玩家点击角色进行标记
8. 切换蓝方(O)工具继续标记
9. 点击结束游戏，确认后返回步骤1

#### 流程2：自选模式
1. 用户访问网站，进入主题选择页面
2. 选择主题（如：明日方舟）
3. 选择网格大小（如：5x5）
4. 选择"自选模式"
5. 系统展示主题内所有角色
6. 用户选择 25 个角色
7. 点击开始游戏
8. 后续流程同随机模式步骤6-9

### 用户体验原则
- **简洁优先**：最少的点击完成操作
- **即时反馈**：所有操作有明确的视觉响应
- **容错设计**：误操作可轻松撤销
- **一致性**：界面风格和交互模式保持统一

### 图片库规范
- **目录结构**：`/public/images/{theme-name}/`
- **文件命名**：使用角色名称（中文或拼音均可）
- **图片格式**：PNG 或 WebP，推荐正方形比例
- **推荐尺寸**：200x200px 至 400x400px
- **元数据**：每个主题目录需包含 `manifest.json`

```json
// /public/images/arknights/manifest.json
{
  "name": "明日方舟",
  "characters": [
    { "id": "amiya", "name": "阿米娅", "image": "amiya.png" },
    { "id": "chen", "name": "陈", "image": "chen.png" }
  ]
}
```

### 约束条件
- **纯静态部署**：无后端服务器，所有逻辑在客户端运行
- **图片托管**：图片存储在仓库内，通过 GitHub Pages 提供
- **浏览器兼容**：支持现代浏览器（Chrome, Firefox, Safari, Edge 最新两个版本）

### 依赖项
- **外部依赖**：无，纯前端应用
- **内部依赖**：图片素材需用户自行准备或后续提供

### 无障碍考虑
- 所有角色图片需有 alt 文本（使用角色名称）
- 标记颜色通过 WCAG AA 对比度标准
- 标记状态有非颜色指示（X/O 符号）
- 支持键盘完整导航（P2 功能）

### 错误场景处理
| 场景 | 处理方式 |
|------|----------|
| 主题角色数量不足 | 禁用超出范围的网格选项，显示提示 |
| 图片加载失败 | 显示占位图，保留角色名称 |
| manifest.json 格式错误 | 控制台警告，跳过该主题 |
| manifest.json 缺失 | 跳过该主题目录 |

## 范围边界

### 包含在内 (v1)
- 主题选择功能
- 随机模式游戏
- 自选模式游戏
- 双方标记系统
- 响应式界面
- GitHub Pages 部署配置
- 明日方舟示例主题（基础角色）

- 在线联机功能
- 用户注册/登录
- 游戏历史记录
- 社交分享功能
- 游戏统计数据

- [ ] **随机选择角色模式**：系统随机选择一个目标角色
- [ ] **单人模式**：与 AI 对战或自我练习
- [ ] **更多主题**：支持其他游戏/动漫角色库
- [ ] **自定义标记样式**：玩家可选择标记颜色和符号
